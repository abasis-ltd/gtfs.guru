Frequently Asked Questions

What is a GTFS validator and why do I need one?

A GTFS validator checks your transit data feed for errors before you submit it to Google Maps, Apple Maps, or other transit apps. Without validation, errors in your GTFS file can cause routes to display incorrectly, schedules to break, or your entire feed to be rejected. GTFS Guru catches these issues instantly, saving hours of debugging.

Is GTFS Guru free to use?

Yes, GTFS Guru is completely free and open source under the Apache 2.0 license. You can use the web validator, download the desktop app, or integrate the Python/Rust libraries into your projects at no cost. There are no usage limits or premium tiers.

Is my transit data safe? Do you store or upload my GTFS files?

Your data never leaves your device. GTFS Guru runs entirely in your browser using WebAssembly (WASM) technology. The desktop app and CLI tools also process everything locally. We don't upload, store, or have access to your transit data—zero bytes are sent to any server.

How is GTFS Guru different from the official Google/MobilityData validator?

GTFS Guru is built in Rust for maximum performance. It validates small feeds up to 100x faster than the canonical Java validator and uses 10x less memory (~150MB vs ~1.5GB). It also offers native Python integration, browser-based validation, and desktop apps—making it easier to integrate into modern workflows.

What GTFS errors can this validator detect?

GTFS Guru checks for all standard GTFS specification errors including: missing required files, invalid field values, broken foreign key references, overlapping trips, impossible travel times, invalid coordinates, calendar inconsistencies, and more. The validator follows the official GTFS reference and MobilityData best practices.

Can I use GTFS Guru in my CI/CD pipeline?

Yes, GTFS Guru is designed for automation. Use the Python package (pip install gtfs-guru) or Rust CLI (cargo install gtfs-guru-cli) in your build pipelines. Both support JSON output for easy parsing, exit codes for pass/fail checks, and can generate HTML reports automatically.

Does GTFS Guru support GTFS-realtime validation?

Currently, GTFS Guru focuses on static GTFS Schedule validation. GTFS-realtime (GTFS-RT) validation for vehicle positions, trip updates, and service alerts is on our roadmap. Join our GitHub to follow development and request features.

How do I add my bus routes to Google Maps?

First, create a valid GTFS feed with your routes, stops, and schedules. Use GTFS Guru to validate it and fix any errors. Then submit your feed through the Google Transit Partner Dashboard. Google reviews submissions and typically publishes approved feeds within 1-4 weeks. A clean, error-free GTFS file speeds up the approval process.

What file format does the validator accept?

GTFS Guru accepts standard GTFS ZIP files containing the required CSV files (agency.txt, stops.txt, routes.txt, trips.txt, stop_times.txt, calendar.txt or calendar_dates.txt). Simply drag and drop your .zip file into the validator or select it using the file browser.

Can I validate GTFS feeds larger than 50MB?

Yes. The browser validator handles most feeds but may be slower for very large files due to browser memory limits. For feeds over 50MB, we recommend using the desktop app or CLI tools, which can process feeds of any size efficiently using your system's full resources.
